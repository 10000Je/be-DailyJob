### 2. 오늘 가장 의미있었던 기술적 판단

- 내가 내린 판단: HttpResponse 생성자를 재활용하는 대신에 2개를 따로 만든 것
- 그렇게 판단한 이유: 기존 코드를 재사용하는 재사용성과 코드의 가독성을 올리는 명시성 중에 어떤 것을 중시할지 고민하는 것이 유의미했던 것 같음.
- 고려했지만 선택하지 않은 대안: HttpResponse 생성자를 오버로드 할 때 기존의 생성자를 재사용하는 방법
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: 생성자에 조건 분기 코드를 별도로 작성하지 않아도 되어서 가독성이 좋아짐

### 3. 가장 막혔던 문제와 해결 과정

- 문제 상황 또는 증상: 로그인이나 회원가입 실패시 실패 메시지를 한글로 할 경우 깨지는 증상
- 처음 추측한 원인: 문자열을 바이트로 바꾸는 과정에서 인코딩을 명시하지 않아 발생한 것으로 추측
- 원인을 확인하기 위해 시도한 방법: 인코딩을 StandardCharsets.UTF_8 로 지정해서 수정
- 실제 원인: 응답에 Content-Type을 명시하지 않은 것, 브라우저가 제대로 해석하지 못해 생긴 일
- 해결 방법: ResponseHandler가 응답을 만드는 책임까지 맡고있던걸 HttpRequest 클래스로 분리,
  이후 content-type을 response의 헤더에 추가하여 해결함
- 다음에 비슷한 문제가 생기면 먼저 확인할 것: 브라우저의 기준에서도 생각할 것

- 그 접근이 도움이 된 이유: 나중에 이를 까먹지 않고 수정할 수 있음
- 다음에도 반복하고 싶은 점: 현재 한계로 인해 지금 당장 수정 못하는 부분은 꼭 표시하는 것

### 5. 오늘 아쉬웠던 행동(개발과 관련된 것)

- 비효율적이었던 접근/놓치거나 부족했던 점: 코드를 짜고나서 직접 돌려보지 않고 AI에게 바로 리뷰를 맡긴 것
- 그렇게 된 원인: 무의시적으로 직접 실행하기 귀찮아서 그랬던 것 같음
- 다음에는 다르게 해볼 구체적인 행동: 코드를 직접 돌려보고 에러가 있는지 없는지 먼저 스스로 판단 후 AI에게 테스트 코드 작성을 맡겨서 테스트 하는 방향으로 나아가볼 것
