# be-DailyJob

> 이 회고는 작업량을 보고하기 위한 문서가 아니라,
> 오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다.
> 각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.

### 1. 오늘 완료한 것

- 오늘 실제로 끝낸 기능/수정/학습: 동적 Html 반환 기능, ResponseHandler 리팩토링
- (오늘의 목표 중에서) 아직 끝내지 못한 부분: index.html 동적 html 구현

### 2. 오늘 가장 의미있었던 기술적 판단

- 내가 내린 판단: 세션/DB에 대한 부수 효과도 Handler 클래스에 응집시킨 것
- 그렇게 판단한 이유: 클래스에 대한 "응집도" 향상과 메소드의 "side-effect" 사이의 Trade-off를
  생각해볼 수 있었던 판단이였음
- 고려했지만 선택하지 않은 대안: side-effect를 최소화하기 위해 RequestHandler에 작성하는 것
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: RequestHandler가 보다 간소화됨

### 3. 가장 막혔던 문제와 해결 과정

- 문제 상황 또는 증상: 301/302 상태코드 버그
- 처음 추측한 원인: HttpResponse 객체를 직접 만든 것
- 원인을 확인하기 위해 시도한 방법: ResponseHandler에 정의된 팩토리 메소드를 호출
- 실제 원인: RequestHandler 비교부분에서 302가 아닌 301로 비교한 것
- 해결 방법: 오타 수정
- 다음에 비슷한 문제가 생기면 먼저 확인할 것: 항상 오타를 확인할 것

### 4. 오늘 잘한 행동(개발과 관련된 것)

- 어떤 행동이었는지: Handler 클래스의 책임을 어디까지 둬야하는지 고민한 것
- 그 접근이 도움이 된 이유: 응집도와 부수효과 사이의 trade-off 를 고민해볼 수 있었음.
- 다음에도 반복하고 싶은 점: 이러한 trade-off를 항상 생각하며 개발하면 좋을 것 같음

### 5. 오늘 아쉬웠던 행동(개발과 관련된 것)

- 비효율적이었던 접근/놓치거나 부족했던 점: 이미 재사용하려고 만든 메소드를 안쓴 것
- 그렇게 된 원인: 만들어두고 까먹음
- 다음에는 다르게 해볼 구체적인 행동: 항상 코드를 재사용할 수 없을까 생각할 것
