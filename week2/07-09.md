### 2. 오늘 가장 의미있었던 기술적 판단

- 내가 내린 판단: RequestHandler의 if-else 구조를 Router를 이용해서 개선한 것
- 그렇게 판단한 이유: 라우팅 포인트가 늘어남에 따라 코드가 길어지는 것을 막을 수 있었음
- 고려했지만 선택하지 않은 대안: RequestHandler에 Map 자료구조를 하나 만들어서 관리하는 것
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: RequestHandler의 라우팅 책임을 분리할 수 있어서 코드가 보기 편해지고
  유지보수성이 향상되었음.

### 3. 가장 막혔던 문제와 해결 과정

- 문제 상황 또는 증상: 로그인 상태에서 로그인/회원가입 페이지 접근시 메인페이지로 리다이렉션 되어야 하는데
  세션이 사라지면서 로그아웃 처리되는 문제
- 처음 추측한 원인: 로그인/로그아웃 GET 엔드포인트 처리시 어딘가에 로그아웃 로직이 숨겨져 있는 것으로 추측
- 원인을 확인하기 위해 시도한 방법: 코드를 확인해봄
- 실제 원인: 세션 내부의 자료구조를 일반 HashMap으로 두어서 여러 스레드가 접근할 때 Race-Condition이 발생, 세션 구조가
  망가지는 것이 원인이었음.
- 해결 방법: 자료구조를 ConcurrentHashMap으로 변경하였음
- 다음에 비슷한 문제가 생기면 먼저 확인할 것: 멀티쓰레드 관련 문제는 단위테스트로도 잘 발견되지 않기 때문에 항상 조심할 것
