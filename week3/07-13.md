# be-DailyJob

> 이 회고는 작업량을 보고하기 위한 문서가 아니라,
> 오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다.
> 각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.

### 1. 오늘 완료한 것

- 오늘 실제로 끝낸 기능/수정/학습: 이미지 업로드 기능 구현, 이미지 보여주기 기능 구현,
  mypage에 프로필 변경 기능 추가, 이전 글/다음 글 기능 추가
- (오늘의 목표 중에서) 아직 끝내지 못한 부분: 댓글 기능, 좋아요 기능, 비밀번호 변경 기능,
  닉네임 변경 기능 구현

### 2. 오늘 가장 의미있었던 기술적 판단

- 내가 내린 판단: /index.html을 리다이렉트에서 → 최신 글 직접 렌더로 바꾼 결정
- 그렇게 판단한 이유: 버그가 없었는데도 "북마크한 URL이 항상 최신을 가리켜야 한다"는 URL 의미의 관점에서 스스로 문제를 짚어, 코드 편의가 아닌 사용자 경험 기준으로 재설계했기 때문
- 고려했지만 선택하지 않은 대안: 기존의 리다이렉팅 방식을 계속 사용하는 것
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: 사용자가 북마크 등으로 페이지를 저장할 때

### 3. 가장 막혔던 문제와 해결 과정

- 문제 상황 또는 증상: 리소스 파일을 수정했는데 테스트에 반영이 안된 것
- 처음 추측한 원인: 소스 코드를 잘못 작성했다고 추측
- 원인을 확인하기 위해 시도한 방법: 소스 코드 파일을 확인
- 실제 원인: 빌드 후 실행할 시 intelliJ는 resources 파일이 아닌 build/ 경로에서 가져옴
- 해결 방법: 재빌드 후 실행하니 테스트에 반영되었음.
- 다음에 비슷한 문제가 생기면 먼저 확인할 것: 빌드 테스트 떄에는 build/ 디렉토리에서 확인할 것

### 4. 오늘 잘한 행동(개발과 관련된 것)

- 어떤 행동이었는지: 설계 결정을 단계별로 명확히 짚고 간 것
- 그 접근이 도움이 된 이유: 과정을 미리 계획하고 단계화하여 진행하여서 실수를 줄일 수 있었음
- 다음에도 반복하고 싶은 점: 단계화

### 5. 오늘 아쉬웠던 행동(개발과 관련된 것)

- 비효율적이었던 접근/놓치거나 부족했던 점: 요구사항을 뒤늦게 추가/변경해서 재작업을 한 것
- 그렇게 된 원인: 단계화 과정에서 실수로 누락함.
- 다음에는 다르게 해볼 구체적인 행동: 요구사항을 꼼꼼히 확인하여 단계화 과정에서 꼭 포함할 것
